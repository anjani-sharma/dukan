# Deploying DOKAN to Render + Vercel + Neon

## Overview

| Layer | Service | Cost |
|---|---|---|
| Database | Neon (PostgreSQL) | Free (0.5 GB) |
| API / Backend | Render (Web Service) | Free (spins down) / $7/mo always-on |
| Frontend | Vercel | Free |

---

## Step 1 — Database on Neon

1. Go to [neon.tech](https://neon.tech) → Sign up → **New Project**
2. Name it `dokan-rk` → choose region **Asia Pacific (Singapore)**
3. Copy the **Connection string** — it looks like:
   ```
   postgresql://user:pass@ep-xxx.ap-southeast-1.aws.neon.tech/neondb?sslmode=require
   ```
4. Keep this — you'll need it in Step 2 and Step 3

### Push the database schema to Neon

Run this once from your local machine (or Replit shell):

```bash
DATABASE_URL="<your-neon-connection-string>" pnpm --filter @workspace/db run push
```

---

## Step 2 — Backend on Render

1. Go to [render.com](https://render.com) → **New Web Service**
2. Connect your GitHub repo (push this project to GitHub first if you haven't)
3. Render will detect `render.yaml` automatically — click **Apply**
4. Set these environment variables in the Render dashboard:
   - `DATABASE_URL` → your Neon connection string
   - `OPENAI_API_KEY` → your OpenAI key
   - `TELEGRAM_BOT_TOKEN` → your Telegram bot token
5. Deploy — note your Render URL: `https://dokan-11zk.onrender.com`

### Register the Telegram webhook (after deploy)

Replace `<render-url>` with your actual Render URL:

```bash
curl "https://api.telegram.org/bot<TELEGRAM_BOT_TOKEN>/setWebhook?url=https://<render-url>/api/telegram/webhook"
```

---

## Step 3 — Frontend on Vercel

1. Go to [vercel.com](https://vercel.com) → **New Project** → Import your GitHub repo
2. Set **Root Directory** to `artifacts/shop-manager`
3. Vercel will detect `vercel.json` and use the correct build command automatically
4. Add this environment variable:
   - `VITE_API_URL` → `https://dokan-11zk.onrender.com`
5. Deploy — your frontend will be live at `https://dokane.vercel.app/`

---

## Step 4 — Point a custom domain (optional)

- **Vercel**: Settings → Domains → Add `dokan.yourdomain.com`
- **Neon**: No change needed
- **Render**: You can also add a custom domain for the API if desired

---

## Replicating for a new client

1. Create a new Neon project for the client (isolated database)
2. Create a new Render service (copy env vars, change `DATABASE_URL`)
3. Create a new Vercel project (copy env vars, change `VITE_API_URL`)
4. Update the branding in `layout.tsx` (client name, logo)
5. Push schema to the new Neon DB
6. Register the Telegram webhook for the new deployment

Each client is fully isolated — separate data, separate URL.

## Environment Variables Reference

### Render (API server)
| Variable | Description |
|---|---|
| `DATABASE_URL` | Neon PostgreSQL connection string |
| `OPENAI_API_KEY` | For voice transcription and invoice scanning |
| `TELEGRAM_BOT_TOKEN` | From @BotFather on Telegram |
| `SESSION_SECRET` | Auto-generated by Render |
| `PORT` | Set to `10000` (Render default) |

### Vercel (Frontend)
| Variable | Description |
|---|---|
| `VITE_API_URL` | Full URL of your Render API, e.g. `https://dokan-11zk.onrender.com` |
